# 📦 Inventory Management System — Three-Tier Application on OpenChoreo

A complete, production-ready **Inventory Management System** built as a three-tier cloud-native application deployed on [OpenChoreo](https://github.com/openchoreo/openchoreo). This example demonstrates how to wire a **Next.js frontend**, an **Express.js backend API**, and a **PostgreSQL database** together using OpenChoreo's component model.

## 🗄️ Step 1 — Provision the PostgreSQL Database (Stateful Component)

The PostgreSQL database is the foundation of the stack and takes the most time to provision. **Start this first.**

### 1.1 — Create the Database via OpenChoreo Console

Navigate to **OpenChoreo Console → Your Project → Stateful → + Create**.

Select:
- **Type:** PostgreSQL
- **Name:** `inventory-db`
- **Region:** Choose the region closest to your users
- **Service Plan:** `startup-4` (recommended for non-production), `business-4` for production

> ⏳ Provisioning takes approximately **5–10 minutes**. Continue with the next steps while you wait.

### 1.2 — Initialize the Schema

Once the database is active, retrieve the connection details from the OpenChoreo Console under **Stateful → inventory-db → Connection Info**.

Run the initialization script against the database:

```bash
psql "postgres://<USER>:<PASSWORD>@<HOST>:<PORT>/<DBNAME>?sslmode=require" \
  -f database/init.sql
```

**`database/init.sql`** — Database Schema:

```sql
-- Enable UUID generation
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Products / Inventory table
CREATE TABLE IF NOT EXISTS inventory (
    id          UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    sku         VARCHAR(100) UNIQUE NOT NULL,
    name        VARCHAR(255) NOT NULL,
    description TEXT,
    quantity    INTEGER NOT NULL DEFAULT 0,
    unit_price  NUMERIC(10, 2) NOT NULL DEFAULT 0.00,
    category    VARCHAR(100),
    location    VARCHAR(100),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Auto-update the updated_at timestamp
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
   NEW.updated_at = NOW();
   RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER set_updated_at
BEFORE UPDATE ON inventory
FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

**`backend/src/routes/inventory.js`**

```javascript
const express = require('express');
const { pool } = require('../db/client');

const router = express.Router();

// GET /api/v1/inventory — list all items (with optional search & category filter)
router.get('/', async (req, res, next) => {
  try {
    const { search, category, page = 1, limit = 20 } = req.query;
    const offset = (page - 1) * limit;

    let query  = 'SELECT * FROM inventory';
    const params = [];
    const conditions = [];

    if (search) {
      params.push(`%${search}%`);
      conditions.push(`(name ILIKE $${params.length} OR sku ILIKE $${params.length})`);
    }
    if (category) {
      params.push(category);
      conditions.push(`category = $${params.length}`);
    }
    if (conditions.length) query += ' WHERE ' + conditions.join(' AND ');
    query += ` ORDER BY created_at DESC LIMIT $${params.length + 1} OFFSET $${params.length + 2}`;
    params.push(limit, offset);

    const { rows } = await pool.query(query, params);
    const countResult = await pool.query('SELECT COUNT(*) FROM inventory');
    res.json({ data: rows, total: parseInt(countResult.rows[0].count, 10) });
  } catch (err) { next(err); }
});

// GET /api/v1/inventory/:id
router.get('/:id', async (req, res, next) => {
  try {
    const { rows } = await pool.query('SELECT * FROM inventory WHERE id = $1', [req.params.id]);
    if (!rows.length) return res.status(404).json({ error: 'Item not found' });
    res.json(rows[0]);
  } catch (err) { next(err); }
});

// POST /api/v1/inventory
router.post('/', async (req, res, next) => {
  try {
    const { sku, name, description, quantity, unit_price, category, location } = req.body;
    const { rows } = await pool.query(
      `INSERT INTO inventory (sku, name, description, quantity, unit_price, category, location)
       VALUES ($1,$2,$3,$4,$5,$6,$7) RETURNING *`,
      [sku, name, description, quantity, unit_price, category, location]
    );
    res.status(201).json(rows[0]);
  } catch (err) { next(err); }
});

// PUT /api/v1/inventory/:id
router.put('/:id', async (req, res, next) => {
  try {
    const { name, description, quantity, unit_price, category, location } = req.body;
    const { rows } = await pool.query(
      `UPDATE inventory SET name=$1,description=$2,quantity=$3,unit_price=$4,category=$5,location=$6
       WHERE id=$7 RETURNING *`,
      [name, description, quantity, unit_price, category, location, req.params.id]
    );
    if (!rows.length) return res.status(404).json({ error: 'Item not found' });
    res.json(rows[0]);
  } catch (err) { next(err); }
});

// DELETE /api/v1/inventory/:id
router.delete('/:id', async (req, res, next) => {
  try {
    const { rowCount } = await pool.query('DELETE FROM inventory WHERE id = $1', [req.params.id]);
    if (!rowCount) return res.status(404).json({ error: 'Item not found' });
    res.status(204).send();
  } catch (err) { next(err); }
});

module.exports = { router };
```

### 2.3 — Configure the Endpoint

After the component is created, OpenChoreo will read the auto-generated `.choreo/component.yaml`. Verify it exposes the HTTP endpoint on port **8080**:

```yaml
# .choreo/component.yaml  (auto-generated — edit minimally if needed)
apiVersion: core.choreo.dev/v1beta1
kind: Component
metadata:
  name: inventory-backend
spec:
  type: Service
  endpoints:
    - name: inventory-api
      port: 8080
      type: REST
      networkVisibility: Project   # Only accessible within the same OpenChoreo project
      context: /api/v1
      schemaFilePath: openapi.yaml # Optional — attach your OpenAPI spec
```

> Set `networkVisibility: Project` so the backend is **only** reachable from within the project (not exposed to the public internet directly). The Next.js frontend reaches it via an internal OpenChoreo Connection.

## 🌐 Step 3 — Build and Configure the Next.js Frontend (Web Application Component)

### 3.1 — Frontend Source Code

**`frontend/lib/api.ts`**

> OpenChoreo injects the backend service URL through `window.configs.apiUrl` when Managed Authentication is enabled. Fall back to the env variable for local development.

```typescript
const getApiBase = (): string => {
  if (typeof window !== 'undefined' && (window as any).configs?.apiUrl) {
    return (window as any).configs.apiUrl;
  }
  return process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8080';
};

export interface InventoryItem {
  id: string;
  sku: string;
  name: string;
  description: string;
  quantity: number;
  unit_price: number;
  category: string;
  location: string;
  created_at: string;
  updated_at: string;
}

export interface InventoryResponse {
  data: InventoryItem[];
  total: number;
}

async function apiFetch<T>(path: string, options?: RequestInit): Promise<T> {
  const res = await fetch(`${getApiBase()}${path}`, {
    ...options,
    credentials: 'include',
    headers: { 'Content-Type': 'application/json', ...options?.headers },
  });
  if (!res.ok) {
    const err = await res.json().catch(() => ({}));
    throw new Error(err.error || `API error: ${res.status}`);
  }
  return res.json();
}

export const inventoryApi = {
  list: (params?: { search?: string; category?: string; page?: number }) =>
    apiFetch<InventoryResponse>(
      `/api/v1/inventory?${new URLSearchParams(params as any).toString()}`
    ),
  get: (id: string) => apiFetch<InventoryItem>(`/api/v1/inventory/${id}`),
  create: (item: Omit<InventoryItem, 'id' | 'created_at' | 'updated_at'>) =>
    apiFetch<InventoryItem>('/api/v1/inventory', {
      method: 'POST',
      body: JSON.stringify(item),
    }),
  update: (id: string, item: Partial<InventoryItem>) =>
    apiFetch<InventoryItem>(`/api/v1/inventory/${id}`, {
      method: 'PUT',
      body: JSON.stringify(item),
    }),
  remove: (id: string) =>
    apiFetch<void>(`/api/v1/inventory/${id}`, { method: 'DELETE' }),
};
```

## 🔐 Step 4 — Configure Managed Authentication (Recommended)

OpenChoreo's Managed Authentication implements a secure **Backend-for-Frontend (BFF)** pattern — tokens are never stored in the browser.

### 4.1 — Sign-in / Sign-out Code

Add the following to your Next.js navigation component:

```tsx
// components/Navbar.tsx
'use client';

export default function Navbar() {
  const handleSignIn  = () => { window.location.href = '/auth/login'; };
  const handleSignOut = () => {
    // session_hint comes from Choreo's injected cookie
    const hint = document.cookie
      .split('; ')
      .find(row => row.startsWith('session_hint='))
      ?.split('=')[1];
    window.location.href = `/auth/logout${hint ? `?session_hint=${hint}` : ''}`;
  };

  return (
    <nav>
      <span>📦 Inventory Management System</span>
      <button onClick={handleSignIn}>Sign In</button>
      <button onClick={handleSignOut}>Sign Out</button>
    </nav>
  );
}
```

### 4.2 — Add a Test User

In the OpenChoreo Console, go to **Project → Manage Test Users → + Add User** and create a test account to log in to the Development environment.

## 🐛 Troubleshooting

| Symptom | Likely Cause | Resolution |
|---------|-------------|------------|
| Backend pod crashes on startup | Missing DB env vars | Verify the database Connection is saved and the backend is re-deployed |
| `ECONNREFUSED` connecting to DB | Wrong hostname/port | Check `CHOREO_INVENTORY_DB_HOSTNAME` — should be the Choreo-injected value, not `localhost` |
| Frontend shows blank screen | Missing `config.js` script tag | Ensure `<script src="/public/config.js" />` is in `layout.tsx` |
| CORS errors in browser | Incorrect network visibility | Set backend endpoint `networkVisibility` to `Project`, not `Public` |
| 404 on `/api/v1/*` from frontend | API base URL not resolved | Check `window.configs.apiUrl` is populated; verify the frontend-backend Connection |
| Build fails with Dockerfile error | Wrong build context | Confirm the **Component Directory** in OpenChoreo points to `frontend/` or `backend/` |
